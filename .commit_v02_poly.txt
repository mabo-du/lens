feat(v0.2): polygon region backend foundation (migration 06 + image_polygons IPC + tests + TS types)

Substantial followup to round-72 v0.1.1 (image-viewer + bbox regions).
This commit lands the **persistence + IPC foundation** for free-form
polygon regions on image documents; the frontend UX
(ImageViewer.tsx polygon mode) lands in a follow-up round.

Schema:
- migration 06_image_polygon.sql: 12-step CREATE TABLE selection_new rebuild
  to expand selection.selection_type CHECK to include 'image_polygon'
  alongside 'text'/'image_region'/'media_ts'. SQLite has no ALTER CONSTRAINT,
  so the proven migration 05 pattern (PRAGMA foreign_keys=OFF -> BEGIN ->
  CREATE selection_new -> INSERT SELECT -> DROP -> ALTER RENAME ->
  recreate 3 secondary indexes -> COMMIT -> PRAGMA foreign_keys=ON) is
  used for cross-version SQLite portability. Then adds the image_polygon
  extension table outside the tx (brand-new table, no rebuild needed).

Backend IPC (image_polygons.rs, mirrors image_regions.rs):
- image_polygon_create: validates 3..64 vertices, each pair = 2 finite f64
  in [0,1]; checks document is image format; transactions on insert
  (selection + image_polygon); links created_by from local_user.
- image_polygon_list_by_document: JOINs selection + image_polygon with
  selection_type='image_polygon' filter; ORDER BY image_polygon.created_at.
- image_polygon_delete: DELETE with selection_type guard so it cannot
  remove bbox/video selections; ON DELETE CASCADE on the FK handles
  the extension row automatically.

Registration:
- migrations.rs: appended ('06_image_polygon', include_str!(...)) tuple.
- commands/mod.rs:  pub mod image_polygons;
- main.rs:          3 new invoke_handler entries between
                    image_regions::image_selection_delete and
                    documents::document_get_asset_base64.

Tests:
- image_selection_polygon_round_trip (tests.rs): seeds project/code/png-doc,
  INSERTs selection_type='image_polygon' row + image_polygon row inside a tx,
  SELECTs via the same JOIN image_polygon_list_by_document uses, deletes the
  parent and asserts the extension row was cascaded.

Frontend IPC (src/ipc/image-polygons.ts):
- ImagePolygonRecord: { id, documentId, codeId, vertices: number[][],
  createdBy | null, createdAt } - vertices typed so the FE gets parsed
  coordinates with no extra JSON.parse work.
- imagePolygonsIpc.create / listByDocument / delete wrappers; create()
  serializes vertices: number[][] -> verticesJson: string at the invoke
  call site (consistent with image-regions.ts spread pattern).

Gates green at commit time:
- tsc: clean
- cargo build + cargo test: 57/57 tests including the new polygon round-trip
- vitest: 14/14
- vite build: clean

Follow-up routable: ImageViewer.tsx polygon mode UX
